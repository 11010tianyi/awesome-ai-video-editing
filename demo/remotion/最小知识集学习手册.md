## 最小知识集

### 概念 1：React 组件 = 视频帧

**大白话**：每一帧画面就是一个 React 组件在某一时刻的渲染结果。你写一个 `<MyScene />` 组件，Remotion 会问：「第 0 帧长什么样？第 30 帧呢？第 60 帧呢？」然后把每帧截图拼成视频。

**为什么核心**：这是 Remotion 最根本的心智模型。理解了这个，其他一切都是这个范式的延伸。

**和其他概念的关系**：由它衍生出 `useCurrentFrame()`（获取当前帧号）和 `interpolate()`（让属性随帧变化）。

**类比**：想象你在画一本翻页动画书。每一页就是一帧。React 组件就是你画每一页的「规则」——告诉计算机「第 1 页圆在左边，第 2 页圆移到中间，第 3 页圆在右边」。

**小练习**：创建一个组件，让一个红色方块在 30 帧内从屏幕左边移动到右边。运行 `npx remotion preview` 看效果。

---

### 概念 2：Composition（合成）

**大白话**：一个 Composition 就是一个「视频片段」的定义——多长、多少帧、什么分辨率、用哪个组件来渲染。

**为什么核心**：没有 Composition，Remotion 不知道你要渲染什么。它是连接 React 组件和最终视频的桥梁。

**和其他概念的关系**：Composition 引用一个 React 组件，设置帧数和 FPS，最终通过 `<Series>` 或 `<Sequence>` 组合成完整视频。

**类比**：Composition 就像导演说「这场戏拍 30 秒，30fps，用 1920x1080，主角是这个场景组件」。

**小练习**：在 `src/Root.tsx` 中注册一个 Composition，设置 duration 为 90 帧（3 秒），播放你的方块移动组件。

---

### 概念 3：useCurrentFrame()

**大白话**：一个 React Hook，返回「现在正在渲染第几帧」。从 0 开始计数。

**为什么核心**：这是让画面动起来的基础。没有它，你只能渲染静态画面。

**和其他概念的关系**：配合 `interpolate()` 使用，把帧号映射到具体的属性值（位置、透明度、颜色等）。

**类比**：`useCurrentFrame()` 就像翻页动画书的页码。你知道现在是第几页，才能决定画什么。

**小练习**：在组件中调用 `useCurrentFrame()`，把当前帧号显示在屏幕左上角作为调试信息。

---

### 概念 4：interpolate()

**大白话**：把一个输入范围的值映射到输出范围。最常用的是把「帧号范围」映射到「属性值范围」。

**为什么核心**：这是让属性随时间变化的核心函数。位置、缩放、透明度、颜色——所有动画都靠它。

**和其他概念的关系**：输入通常是 `useCurrentFrame()` 的值，输出是 CSS 属性值。

**语法**：
```typescript
interpolate(
  frame,           // 当前帧号
  [0, 30],         // 输入范围（帧）
  [0, 500],        // 输出范围（像素）
  { extrapolateLeft: 'clamp', extrapolateRight: 'clamp' }
)
```

**类比**：interpolate 就像调音台的推子。推子从 0 推到 10（帧范围），音量从静音变成最大（属性范围）。中间任何位置都能精确对应。

**小练习**：用 `interpolate` 让方块在 0-30 帧从左移到右，30-60 帧从透明变不透明，60-90 帧缩小到 50%。

---

### 概念 5：Sequence 和 Series

**大白话**：
- `<Sequence>`：从某一帧开始，播放一段子组件，持续指定帧数。
- `<Series>`：多个 Sequence 按顺序串联，前一个结束下一个开始。

**为什么核心**：真实视频由多个场景组成。Sequence/Series 是组织场景的基本方式。

**和其他概念的关系**：它们内部的子组件也可以用 `useCurrentFrame()` 获取「本段内的相对帧号」。

**类比**：
- Sequence 像磁带上的一个音轨，从某个时间点开始播放。
- Series 像把多个音轨首尾相连串成一张专辑。

**小练习**：用 Series 把三个场景串起来：场景 1 显示标题（0-60 帧），场景 2 显示图表（60-150 帧），场景 3 显示结尾（150-180 帧）。

---

### 概念 6：useVideoConfig()

**大白话**：返回当前 Composition 的配置信息：帧率（FPS）、宽度、高度、总帧数、持续时长。

**为什么核心**：写可复用模板时，你不能硬编码尺寸和时长，需要从配置中读取。

**和其他概念的关系**：配合 `interpolate()` 计算百分比位置，配合 CSS 实现响应式布局。

**类比**：就像拿到一张纸的尺寸信息（A4 还是 B5），你才能决定画多大、画在哪。

**小练习**：在组件中调用 `useVideoConfig()`，把 width/height/FPS 显示在屏幕上。

---

### 概念 7：样式与动画（CSS + interpolate）

**大白话**：Remotion 组件用标准 React 内联样式。所有 CSS 属性都可以通过 `interpolate()` 动态化。

**为什么核心**：最终画面质量取决于你对 CSS 的掌控。Remotion 不提供专门的动画 API，它信任 CSS。

**常用可动画化的属性**：
- `transform`: `translateX()`, `scale()`, `rotate()`
- `opacity`: 透明度
- `color`, `backgroundColor`: 颜色变化
- `fontSize`, `width`, `height`: 尺寸变化

**类比**：CSS 就是画家的颜料和画笔。interpolate 决定「什么时候用什么颜色」，CSS 决定「画出来是什么效果」。

**小练习**：用 `interpolate` 控制 `rotate` 属性，让一个方块在 90 帧内旋转 360 度。

---

### 概念 8：Remotion CLI

**大白话**：命令行工具，用于预览（`preview`）、渲染（`render`）、升级（`upgrade`）。

**为什么核心**：开发循环完全依赖 CLI。不会用 CLI 寸步难行。

**常用命令**：
```bash
npx remotion preview          # 启动预览，实时看效果
npx remotion render <comp>    # 渲染为 MP4
npx remotion upgrade          # 升级 Remotion 版本
```

**类比**：CLI 就像视频编辑软件的「播放」「导出」「更新」按钮，只是用命令行操作。

**小练习**：运行 `npx remotion preview`，在浏览器中确认你的组件能正常播放。

## 检验问题

### 概念理解

1. **Remotion 中，一帧画面是怎么产生的？**
   > 一个 React 组件在某一帧号下的渲染结果。Remotion 调用组件，传入当前帧号，拿到 JSX 输出，截图。

2. `useCurrentFrame()` 和 `useVideoConfig()` 有什么区别？
   > `useCurrentFrame()` 返回当前帧号（0, 1, 2...），用于控制动画进度。`useVideoConfig()` 返回视频配置（FPS、宽高、总帧数），用于计算和响应式布局。

3. `interpolate(frame, [0, 30], [0, 500])` 在第 15 帧时返回什么？
   > 250。线性插值，15 是 0-30 的中点，所以输出是 0-500 的中点 250。

4. `<Sequence from={30} durationInFrames={60}>` 中的组件，在第 45 帧时 `useCurrentFrame()` 返回什么？
   > 15。Sequence 内部的帧号是相对的，从 0 开始。45 - 30 = 15。

5. 为什么说「Remotion 不提供动画 API」？
   > 因为动画完全通过 CSS 属性 + `interpolate()` 实现。Remotion 只负责「在每一帧调用你的组件」，动画效果由你用标准 CSS 控制。

### 实践判断

6. 如果 `interpolate` 的输入范围是 `[0, 100]`，但当前帧是 150，默认会发生什么？
   > 会继续外推（extrapolate），输出超过目标值。加 `{ extrapolateRight: 'clamp' }` 可以限制在最大值。

7. 想让一个元素在视频播放到一半时出现，用什么方式？
   > 用 `<Sequence from={halfFrame}>` 包裹，或用 `interpolate` 控制 `opacity` 在中间帧从 0 变 1。

8. Remotion 渲染视频时，是实时录制屏幕吗？
   > 不是。它在 Node.js 环境中逐帧调用 React 组件，截图保存为图片序列，再用 FFmpeg 合成视频。所以渲染速度取决于组件复杂度和硬件。
